# i18n (Multi-Language Support) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add English/Dutch multi-language support using react-i18next, with per-user locale stored server-side and translated email templates.

**Architecture:** Frontend uses react-i18next with bundled JSON translation files (no HTTP loading). Backend stores user's locale preference in the `users` table and includes it in auth responses. Email templates accept a locale parameter and use a simple translation map.

**Tech Stack:** react-i18next, i18next, Drizzle ORM (SQLite), Express

**Step 3: Include `locale` in auth service responses**

In `server/src/services/auth.ts`, every function that builds an `authUser` object needs `locale` added. There are 4 places:

In `refreshAccessToken()` (~line 73-78): same pattern, add `locale: user.locale`.

In `registerWithInvitation()` (~line 215-220):


**Step 4: Add locale endpoint and update auth routes**

In `server/src/routes/auth.ts`, add a `PATCH /locale` endpoint after the password change route (~line 184):

```typescript
// Change locale
authRoutes.patch('/locale', authenticate, requireSetupComplete, async (req: Request, res: Response, next: NextFunction) => {
  try {
    const { locale } = req.body;
    if (!locale || !['en', 'nl'].includes(locale)) {
      res.status(400).json({ error: 'Invalid locale. Supported: en, nl' });
      return;
    }

    const now = new Date().toISOString();
    db.update(schema.users)
      .set({ locale, updatedAt: now })
      .where(eq(schema.users.id, req.user!.id))
      .run();

    res.json({ ok: true, locale });
  } catch (err) {
    next(err);
  }
});
```

Also update the username change handler (~line 143) to include `locale` in the authUser:
```typescript
const authUser = { id: user.id, username: newUsername, isAdmin: user.isAdmin, isTemporary: user.isTemporary, locale: user.locale };
```


**Step 2: Create i18n initialization**

Create `client/src/i18n/index.ts`:

```typescript
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import en from './en.json';
import nl from './nl.json';

i18n.use(initReactI18next).init({
  resources: {
    en: { translation: en },
    nl: { translation: nl },
  },
  lng: localStorage.getItem('locale') || 'en',
  fallbackLng: 'en',
  interpolation: {
    escapeValue: false,
  },
});

export default i18n;
```

**Step 3: Import i18n in main.tsx**

In `client/src/main.tsx`, add import before the React render (line 1 area):

